# PROLEUM Monopoly

Цифровой стенд для тестирования настольной игры «ПРОЛЕУМ: Сделка на миллион тонн».

Проект нужен не как финальная коммерческая платформа, а как быстрый мультиплеерный прототип: менять карточки, проверять баланс, гонять партии без перепечатки компонентов.

## Запуск

```bash
npm run dev
```

Локальный адрес:

```text
http://127.0.0.1:5173/
```

## Что уже реализовано

- вход с ролями администратора и игрока;
- комнаты для тестовых партий;
- синхронизация комнат через Supabase API;
- поле на 40 клеток;
- рынок ресурсов REG / PRM / DTL с ценами и остатками дня;
- логистика дня: ЖД и нефтебаза с общей мощностью;
- бросок 2D6 и модальное окно «Ваш ход»;
- ограничение действий: один бросок, одно действие клетки, одно коммерческое действие;
- контракты нельзя закрыть без ресурсов;
- поставка закрывается через выбор маршрута;
- маршрут списывает мощность и запускает проверку риска;
- актив клетки нельзя купить повторно и нельзя купить не стоя на клетке;
- один новый актив за круг;
- сроки контрактов уменьшаются при завершении хода;
- сорванные контракты штрафуют репутацию и победные очки;
- админ-редактор игровых сущностей.

## Supabase

Локально используются публичные переменные:

```text
NEXT_PUBLIC_SUPABASE_URL
NEXT_PUBLIC_SUPABASE_PUBLISHABLE_KEY
```

Состояние комнаты хранится в `public.game_rooms.state` как JSON. Это ускоряет тестирование правил. Для следующего этапа нужен серверный endpoint действий, который будет атомарно валидировать ходы и писать append-only события.
